# 崩溃恢复

**定义**：MySQL 异常重启后，InnoDB 利用 redo log 把已提交但尚未刷入数据文件的操作重做，使数据不丢；并结合两阶段提交与 binlog 判断 redo 中 prepare 状态的事务应提交还是回滚。

**机制**：
- **redo 重做**：按 redo log 顺序重放「在某个数据页上做的修改」，把未落盘的数据页恢复到崩溃前状态。
- **与 binlog 配合**：redo 里 prepare 未 commit 的事务，若对应 binlog 已完整写入则提交，否则回滚，保证与 [[两阶段提交]] 语义一致。
- **恢复流程**：重启后先通过 redo 恢复数据页，再根据 redo+binlog 判定 prepare 事务的提交/回滚。

**语境**：理解「为什么说 InnoDB 有 crash-safe」「误删后如何用全量备份+binlog 恢复到某一秒」时，需结合 redo、binlog 与两阶段提交。
