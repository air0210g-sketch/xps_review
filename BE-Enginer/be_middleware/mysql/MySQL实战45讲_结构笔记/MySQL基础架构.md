# MySQL基础架构

**定义**：MySQL 在逻辑上分为 **Server 层** 与 **存储引擎层**。一条 SQL（查询或更新）先经 Server 的连接器→查询缓存→分析器→优化器→执行器，再由执行器通过引擎接口访问存储引擎读写数据。

**机制**：
- **Server 层**：连接器（建连、鉴权、维持连接；权限在连接建立时读入，改权限不影响已存在连接）、查询缓存（key=SQL，value=结果；表一更新即失效，MySQL 8.0 已移除）、分析器（词法+语法，报错如 "Unknown column 'k'" 在此阶段）、优化器（选索引、选 join 顺序）、执行器（查权限、调用引擎接口取/写行）。内置函数、存储过程、触发器、视图等跨引擎能力都在此层。
- **存储引擎层**：插件式，负责数据存储与提取。默认 InnoDB（MySQL 5.5.5+）。建表不写 `engine=` 即用 InnoDB；可指定 `engine=memory` 等。
- **执行器与引擎**：执行器按「取满足条件的第一行 / 下一行」循环调用引擎；无索引时引擎全表扫描；慢查询日志中的 `rows_examined` 为执行器每次取行累加，与引擎内部扫描行数未必一致。

**语境**：排查「SQL 为什么慢」「报错在哪个阶段」时，先区分是连接/权限、解析、优化选索引还是引擎执行；理解「谁在做什么」才能定位到具体环节。
