# 事务隔离级别

**定义**：SQL 标准下四种隔离级别——读未提交、读提交、可重复读、串行化。MySQL InnoDB 默认 **可重复读**；通过「视图」与多版本在事务内提供一致性读，避免脏读、不可重复读（可重复读级别下），幻读则通过间隙锁等处理。

**机制**：不同级别下，事务何时看到其他事务的修改、是否出现脏读/不可重复读/幻读，由「视图」创建时机与读方式（快照读 vs 当前读）共同决定。与 [[一致性读与当前读]]、[[行锁]] 串联后，可解释「为什么你改了我还看不见」以及加锁范围。

**语境**：排查「同一行在不同事务里看到不同值」「幻读」时，先确认隔离级别与读类型（快照读/当前读）。
