# 行锁

**定义**：InnoDB 在**行级别**加的锁，由引擎层实现。当前读（如 UPDATE、SELECT FOR UPDATE）会对匹配行加行锁；锁在事务提交或回滚时释放。与 [[间隙锁]]、[[next-key lock]] 一起解释「只改一行却锁很多」的现象。

**机制**：两阶段锁协议：行锁在需要时加、在事务结束时释放；因此长事务会持有锁更久，易造成锁等待与死锁。减少锁影响：缩短事务、控制并发顺序。

**语境**：慢查询、锁等待、死锁排查时，结合事务隔离级别与当前读/加锁规则分析。
