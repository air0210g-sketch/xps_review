# 全局锁与表锁

**定义**：按加锁范围，MySQL 有全局锁（整库只读）、表级锁（表锁与 MDL）。与行锁相对，锁粒度大，影响并发。

**机制**：
- **全局锁**：`Flush tables with read lock (FTWRL)` 使整库只读，阻塞 DML、DDL 及更新类事务提交。典型用途是全库逻辑备份；全 InnoDB 库可用 `mysqldump --single-transaction` 替代（事务内一致性读，不阻塞写）。不用 `set global readonly=true` 的原因：readonly 常被用于主备判断，且异常断开后不会自动释放。
- **表锁**：`lock tables t1 read, t2 write` 显式加表级读/写锁，并限制本线程只能读 t1、读写 t2；`unlock tables` 或断开释放。InnoDB 一般用行锁，少用 lock tables。
- **MDL（元数据锁）**：访问表时自动加 MDL 读锁，DDL 时加 MDL 写锁；读读不互斥，读写/写写互斥。**事务提交后才释放**；长事务不提交会占着 MDL 读锁，后续 DDL（如加字段）会阻塞，且会阻塞后续所有对该表的 MDL 读锁申请，导致表不可读写。小表加字段导致库挂的常见原因。

**语境**：全库备份选 FTWRL 还是 single-transaction；为何「给表加个字段」会拖垮库（长事务+MDL）；安全加字段需先查长事务、必要时用 WAIT n/NOWAIT（MariaDB/AliSQL）。
