# 一致性读与当前读

**定义**：**一致性读**（快照读）指事务内根据启动时的视图读取数据，不读其他事务未提交或已提交但晚于本事务的修改。**当前读** 指读「当前已提交的最新值」并可能加锁（如 `SELECT ... FOR UPDATE`、`UPDATE`/`DELETE` 所读到的行），会看到其他事务已提交的修改并与 [[行锁]]、[[间隙锁]] 关联。

**机制**：可重复读下，普通 SELECT 是一致性读；UPDATE/INSERT/DELETE 或 SELECT FOR UPDATE 是当前读。专栏 08 讲重点：把一致性读、当前读和行锁串起来，可解释「改了我还看不见」以及「只改一行却锁很多」的锁范围。

**语境**：分析锁与可见性时，先区分该语句是快照读还是当前读。
