# 索引（B+树与InnoDB）

**定义**：InnoDB 使用 **B+ 树** 索引；表数据按主键聚簇存储（聚簇索引），二级索引叶子存主键值。**覆盖索引** 指查询列均在索引中，无需回表；**最左前缀** 指联合索引从左到右匹配，用于解释索引选择与失效。

**机制**：主键查找走聚簇索引；二级索引先查索引再回表（除非覆盖）。索引选择错误、字符串索引方式、order by / count 是否走索引等，都在专栏 04–05、09–18 讲展开。

**语境**：建表与建索引时考虑查询条件与顺序；explain 看是否走索引、是否回表。
