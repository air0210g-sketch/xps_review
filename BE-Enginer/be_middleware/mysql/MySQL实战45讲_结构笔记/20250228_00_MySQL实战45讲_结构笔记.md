# MySQL实战45讲 · 结构笔记

## 核心命题

**理解 MySQL 的原理（从一条 SQL 的执行路径到事务、索引、锁、主备），才能在实践中做出正确选择、快速定位问题，并形成「点→线→网→面」的持久知识网络。**

---

## 逻辑支撑链

1. **原理决定行为**：SQL 如何执行、事务如何隔离、索引如何被选用、锁如何加，都由 MySQL 内核机制决定；不知道原理就无法解释「为什么这条 SQL 慢」「为什么只改一行却锁很多」。
2. **行为决定实践**：写表结构、建索引、写 SQL、做主备与扩容，每一步都是对上述原理的应用；原理清晰后，最佳实践（如尽量长连接、避免查询缓存、主键自增）才有「为什么」的支撑。
3. **实践反哺网络**：通过动手复现、讲出来/写出来、把点连成线再交叉成网（结束语），知识才能持久；专栏的目标是充当这一过程的加速器。

---

## 整体结构（三层）

- **第一层：基础架构与一条 SQL 的生命周期**  
  从连接器到执行器（Server 层）+ 存储引擎层；查询与更新在日志（redo/undo/binlog）上的差异。  
  → 对应 01–02 讲，及后续各讲中反复出现的「执行器/引擎接口」视角。

- **第二层：三条主线——事务、索引、锁**  
  - **事务**：隔离级别、可见性、一致性读 vs 当前读、redo/undo 与崩溃恢复（03, 08, 23 等）。  
  - **索引**：数据结构、普通/唯一索引选择、字符串索引、选错索引、表空间与 count/order by（04–05, 09–11, 13–18）。  
  - **锁**：全局锁/表锁/行锁、间隙锁与 next-key lock、死锁与锁等待（06–07, 19–22, 30）。  

- **第三层：高可用与综合实践**  
  主备一致、binlog 格式、延迟与切换、读写分离、误删与 kill、临时表、join、内存与分区等（23–45）。
