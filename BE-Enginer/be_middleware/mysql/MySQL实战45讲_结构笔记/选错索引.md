# 选错索引

**定义**：优化器在可选索引中选错了执行计划，导致本该走索引的查询变成全表扫描或走了代价更高的索引，执行变慢。

**机制**：
- **依据**：优化器根据扫描行数（含索引基数 cardinality）、是否排序、是否用临时表等估代价；扫描行数来自索引统计（采样统计，N 个数据页取样，易不准）。
- **典型原因**：删除/插入导致索引统计滞后；或优化器高估了「走二级索引+回表」的代价，选了全表扫描。
- **处理**：① `analyze table t` 重新统计索引信息；② `force index(a)` 强制用某索引（不优雅、改索引名要改 SQL）；③ 改 SQL 引导优化器（如 order by b,a limit 1 使扫描行数成主要因素）；④ 删掉误用索引或加更合适索引。

**语境**：explain 显示 key 与预期不符、rows 异常大时，先看是否统计不准（analyze），再考虑 force index 或改语句。
