# 间隙锁

**定义**：InnoDB 在**索引记录间隙**上加的锁，用于在可重复读级别下防止幻读。例如表中 id 有 1,5,10，则间隙包括 (1,5)、(5,10) 等；对某范围加锁时，会锁住范围内的行与间隙，导致「只改一行却锁住一个区间」。

**机制**：与 [[next-key lock]] 关系：next-key lock = 行锁 + 该行前的间隙锁。间隙锁之间不互斥（可共存），但会阻塞「向间隙插入」的操作，从而避免幻读。

**语境**：解释「为什么只改一行锁这么多」、死锁与锁等待时，需看是否涉及间隙锁与 next-key lock。
