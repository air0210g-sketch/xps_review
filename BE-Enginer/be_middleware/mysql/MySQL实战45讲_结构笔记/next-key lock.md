# next-key lock

**定义**：InnoDB 在可重复读下的**加锁单位**，由 **行锁** + **该行之前的间隙锁** 组成，即左开右闭区间。例如索引有 1,5,10，对 5 加 next-key lock 会锁 (1,5] 及间隙 (1,5)，从而锁住一个区间。

**机制**：加锁规则（唯一索引/普通索引、等值/范围）决定 next-key 的退化为行锁或间隙锁。与 [[行锁]]、[[间隙锁]] 串联可完整解释「只改一行的语句锁很多」；也是分析死锁与锁等待的常用概念。

**语境**：排查锁范围、死锁、阻塞时，结合加锁顺序与 next-key 退化为行锁或间隙锁的规则。
