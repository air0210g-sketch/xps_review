# 主从复制（Replication）

## Core Concept

**主从复制**是 Redis 将**主节点（master）**的内存数据同步到**从节点（replica/slave）**的机制，用于读写分离、备份与高可用基础；从库只读，写只在主库，主库把写命令或快照发给从库。

*例如：一主两从时，主库写入 set k v，两个从库会收到相同写流或先全量 RDB 再增量，最终与主库数据一致（存在复制延迟）。*

## Mechanism

*   **全量同步**：从库首次连接或复制偏移量落后太多时，主库 fork 生成 RDB 发给从库，从库清空后加载 RDB，再应用期间的增量命令。
*   **增量同步**：主库把写命令写入 **repl_backlog**（环形缓冲区），从库断线重连后发自己的复制偏移量，主库从 backlog 中补发缺失段；若 backlog 被覆盖则退化为全量。
*   **复制偏移量**：主从各自维护 offset，主库每写一个字节从库也推进，用于断线后「从哪继续」；主库的 repl_backlog_size 决定可容忍的断线时长。
