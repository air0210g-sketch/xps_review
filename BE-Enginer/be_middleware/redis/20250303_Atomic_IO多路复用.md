# IO 多路复用（I/O Multiplexing）

## Core Concept

**IO 多路复用**是在**单线程/单进程**内用**一个「等待句柄」**同时监听多个 IO 描述符（如 socket），在**任一**描述符就绪（可读/可写）时返回，由程序再对就绪描述符做读写，从而用少量线程服务大量连接。

*例如：Redis 主线程通过 epoll 监听成百上千个客户端 socket，谁先发来数据谁先被处理，而不必为每个连接开一个线程阻塞在 read 上。*

## Mechanism

*   **本质**：把「等 IO」从「占着 CPU 等」变成「交给内核等，就绪后通知」。内核维护就绪集合，应用只做「注册 → 等待 → 就绪后读写」。
*   **常见实现**：select / poll（轮询就绪）、epoll（Linux，事件驱动，O(1) 就绪）、kqueue（BSD/macOS）。Redis 在 Linux 上用 **epoll**。
*   **与单线程的关系**：多路复用解决「多连接谁先就绪」；单线程顺序执行命令解决「无锁、无竞态」。二者结合 = 单线程 + 多连接并发 IO。
