# 哨兵（Sentinel）

## Core Concept

**哨兵**是 Redis 的**高可用**组件：监控主从节点，在主节点故障时**自动选举新主**并让从库与客户端指向新主，实现故障转移（failover）；通常多实例部署（如 3 个哨兵）以自身高可用。

*例如：一主两从 + 三哨兵，主宕机后哨兵达成「客观下线」并选出一个从提升为主，其余从指向新主，客户端通过哨兵拿到新主地址。*

## Mechanism

*   **主观/客观下线**：单个哨兵认为主 PING 超时为「主观下线」；达到 quorum 个哨兵都认为主下线则为「客观下线」，才触发选举。
*   **选举与切换**：哨兵之间用类似 Raft 的选举选出一个 leader，由它执行故障转移：在从库中选新主（规则：优先级、复制偏移、runid）、执行 SLAVEOF NO ONE、让其他从指向新主、通知客户端。
*   **配置纪元**：每次故障转移递增，用于区分不同「轮次」的配置，避免旧配置覆盖新配置。
