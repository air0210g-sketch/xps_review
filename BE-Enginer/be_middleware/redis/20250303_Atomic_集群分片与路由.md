# 集群分片与路由（Cluster Sharding & Routing）

## Core Concept

**Redis 集群**把数据按 **16384 个哈希槽（slot）** 分片到多个节点，每个 key 用 CRC16(key) mod 16384 落在某槽，槽分布在不同实例上；客户端或代理根据槽定位到正确节点，实现水平扩展与高可用（每槽可带从库）。

*例如：3 主 3 从，槽 0–5460 在节点 A，5461–10922 在 B，10923–16383 在 C；执行 set k v 时先算槽，再发往对应节点。*

## Mechanism

*   **哈希槽**：固定 16384 槽，key 用 CRC16(key) % 16384 得槽号；槽与节点的映射在集群内通过 Gossip 传播，每个节点持有一份。
*   **MOVED 与 ASK**：请求发错节点时，节点若已迁移完该槽则回 **MOVED slot ip:port**，客户端更新本地映射并重定向；若槽正在迁移则回 **ASK**，客户端只对当次请求发 ASKING 后访问目标节点。
*   **扩容与迁移**：新节点加入后，从现有节点迁入部分槽；迁移过程中槽内 key 逐个从源迁到目标，请求先查源，未命中再 ASK 到目标。
