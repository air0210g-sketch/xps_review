# 分布式锁（Distributed Lock）

## Core Concept

**分布式锁**是在多进程/多机环境下，用**同一份共享存储**（如 Redis）实现**互斥**：同一时刻只有一个客户端能「持有」锁，从而串行化对共享资源的访问；常用 SET key unique_value NX PX ttl 实现「占位 + 过期」。

*例如：多台应用服务器同时扣库存，先向 Redis 申请锁 key=lock:stock:101，拿到锁的才能执行扣减，执行完 DEL 或 Lua 释放；未拿到则重试或快速失败。*

## Mechanism

*   **唯一值**：加锁时 value 设为唯一标识（如 UUID），释放时只删「自己加的锁」（先 GET 再 DEL 非原子，需用 Lua 或 Redis 6.2+ 的 GETDEL + 比较），避免误删他人锁。
*   **过期时间**：必须设 PX/EX，否则持锁方宕机会导致死锁；过期时间需大于业务执行时间，否则业务未完成锁失效，其他客户端会重复获取。
*   **续期（watchdog）**：长任务可在持有期内定时续期（如每 ttl/3 执行 PEXPIRE），避免因执行超时导致锁过期被他人抢占；Redisson 等库内置该逻辑。
*   **Redlock**：多 Redis 实例（主从独立）多数派加锁，降低单点故障导致的双写；存在时钟与网络假设争议，需结合场景评估。
